36/**
37\brief Easy loading and saving of human-readable configuration files.
38
39Any encodes typed, structured data and can serialize it to a human
40readable format that is very similar to the Python language's data
41syntax, and fully supports Python's data syntax as well.
42It is well-suited for quickly creating human-readable file
43formats, especially since deserialization and serialization preserve
44comments and an Any can tell you what file and line it came from. The
45syntax allows most C++ editors to properly highlight Any files, and
46makes it easy to design little ad-hoc C-like languages in
47configuration files.
48
49The class is designed so that copying Anys generally is fast, even if
50it is a large array or table. This is because data is shared between
51copies until it is mutated, at which point an actual copy occurs.
52
53\section Example
54Sample File:
55<pre>
56{
57 shape = "round",
58
59 // in meters
60 radius = 3.7,
61
62 position = Vector3(1.0, -1.0, 0.0),
63 video = { format = "RGB8", size = (320, 200)},
64
65 material = \#include("rocks.mat")
66}
67</pre>
68
69Sample code using Any:
70<pre>
71Any x;
72x.load("ball.txt");
73if (x["shape"].string() == "round") {
74 x["density"] = 3;
75}
76x.save("ball.txt");
77</pre>
78
79The custom serialization format was chosen to be terse, easy for
80humans to read, and easy for machines to parse. It was specifically
81chosen over formats like XML, YAML, JSON, S-expressions, and Protocol
82Buffers, although there is no reason you could not write readers and
83writers for G3D::Any that support those. Any also currently supports
84the JSON format.
85
86G3D::Any assumes that structures do not contain cycles; it is an
87error to create a structure like:
88
89<pre>
90Any x(Any::ARRAY);
91x.array().append(x); // don't do this!
92</pre>
93
94although no exception will be thrown at runtime during that append.
95
96\section includes
97
98When parsing an Any from a file, the syntax
99<code>\#include(<i>filename</i>)</code> allows subsitution of the contents of
100<i>filename</i> for any single expression in an Any. The filename is interpreted
101relative to the file being parsed, and inside of the included file, relative filenames
102are interpreted with respect to the included file (and so on, recursively for nested
103inclusion).
104
105Filenames are resolved with System::resolve and then System::findDataFile if not found,
106so they may contain environment variables.
107
108\section Parsing
109
110The primary use of Any is to create your own text file formats.
111The Vector3 constructor is a good example of how to use the Any::verify
112methods to provide good error checking while parsing such formats:
113
114<pre>
115Vector3::Vector3(const Any& any) {
116 any.verifyName("Vector3");
117 any.verifyType(Any::TABLE, Any::ARRAY);
118 any.verifySize(3);
119
120 if (any.type() == Any::ARRAY) {
121 x = any[0];
122 y = any[1];
123 z = any[2];
124 } else {
125 // Table
126 x = any["x"];
127 y = any["y"];
128 z = any["z"];
129 }
130}
131</pre>
132
133It is often convenient to iterate through the table portion using G3D::AnyTableReader.
134
135\section BNF
136Serialized format BNF:
137
138\htmlonly
139<pre>
140identifier ::= (letter | "_") (letter | digit | "_")*
141identifier-op ::= "::" | "->" | "."
142
143identifier-exp ::= [identifier-op] identifier (identifier-op identifier)*
144
145comment ::= C++ single or multi-line comments
146separator ::= "," | ";"
147
148number ::= <legal C printf number format>
149string ::= <legal C double-quoted or unquoted string; backslashes must be escaped>
150boolean ::= "True" | "False"
151nil ::= "Nil" <not case sensitive> | "None" <case sensitive>
152array ::= ("(" | "[" | "{") [value (separator value)* [separator] ] (")" | "]" | "}")
153pair ::= (identifier | string) ("=" | ":") value
154table ::= ("(" | "[" | "{") [ pair (separator pair)* [separator] ] (")" | "]" | "}")
155named-array ::= identifier-exp array
156named-table ::= identifier-exp table
157include ::= "#" "include" "(" string ")"
158
159value ::= [comment] (nil | number | boolean | string | array | table | named-array | named-table | include)
160</pre>
161\endhtmlonly
162
163Except for single-line comments, whitespace is not significant.
164All parsing is case-insensitive.
165
166The include expression pastes the contents of the named file in as if
167they appeared in the original source. Note that an include expression
168can only appear in the locations where a value is expected. This means
169that it cannot yield more than one element of an array and cannot serve
170as the pair in a table.
171
172The deserializer allows the substitution of [] or {} for () when writing
173tuples and ";" for ",". These are convenient when mimicing a
174programming language, e.g., <code>"[ printf("hello world."); clearScreen();]"</code>
175parses as an array containing two named arrays within it. The
176deserializer also allows a trailing comma inside any array or table,
177which also convenient when commenting out the last element.
178
179The serializer indents four spaces for each level of nesting.
180Tables are written with the keys in alphabetic order.
181
182\sa G3D::AnyTableReader
183*/
