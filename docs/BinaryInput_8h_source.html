49/**
50 Sequential or random access byte-order independent binary file access.
51 Files compressed with zlib and beginning with an unsigned 32-bit int
52 size are transparently decompressed when the compressed = true flag is
53 specified to the constructor.
54
55 For every readX method there are also versions that operate on a whole
56 Array, std::vector, or C-array. e.g. readFloat32(Array<float32>& array, n)
57 These methods resize the array or std::vector to the appropriate size
58 before reading. For a C-array, they require the pointer to reference
59 a memory block at least large enough to hold <I>n</I> elements.
60
61 Most classes define serialize/deserialize methods that use BinaryInput,
62 BinaryOutput, TextInput, and TextOutput. There are text serializer
63 functions for primitive types (e.g. int, std::string, float, double) but not
64 binary serializers-- you <B>must</b> call the BinaryInput::readInt32 or
65 other appropriate function. This is because it would be very hard to
66 debug the error sequence: <CODE>serialize(1.0, bo); ... float f; deserialize(f, bi);</CODE>
67 in which a double is serialized and then deserialized as a float.
68 */
