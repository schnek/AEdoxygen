20/**
21 OS-independent file system layer that optimizes the performance
22 of queries by caching and prefetching.
23
24 This class uses the following definitions:
25 <ul>
26 <li> "file" = document that can be opened for reading or writing
27 <li> "directory" = folder containing files and other directories
28 <li> "node" = file or directory
29 <li> "path" = string identifying a (see the FSPath class)
30 <li> "zipfile" = a compressed file storing an archive of files and directories in the zip format
31 </ul>
32
33 In G3D, Zipfiles are transparently treated as if they were directories, provided:
34 <ul>
35 <li> The zipfile name contains an extension (e.g., map.pk3, files.zip)
36 <li> There are no nested zipfiles
37 </ul>
38
39 All FileSystem routines invoke FilePath::expandEnvironmentVariables if the input contains a '$'.
40
41 The extension requirement allows G3D to quickly identify whether a path could enter a
42 zipfile without forcing it to open all parent directories for reading.
43
44 \sa FilePath
45*/

512/** \brief Parsing of file system paths.
