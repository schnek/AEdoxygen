21/**
22 N x M matrix.
23
24 The actual data is tracked internally by a reference counted pointer;
25 it is efficient to pass and assign Matrix objects because no data is actually copied.
26 This avoids the headache of pointers and allows natural math notation:
27
28 <PRE>
29 Matrix A, B, C;
30 // ...
31
32 C = A * f(B);
33 C = C.inverse();
34
35 A = Matrix::identity(4);
36 C = A;
37 C.set(0, 0, 2.0); // Triggers a copy of the data so that A remains unchanged.
38
39 // etc.
40
41 </PRE>
42
43 The Matrix::debugNumCopyOps and Matrix::debugNumAllocOps counters
44 increment every time an operation forces the copy and allocation of matrices. You
45 can use these to detect slow operations when efficiency is a major concern.
46
47 Some methods accept an output argument instead of returning a value. For example,
48 <CODE>A = B.transpose()</CODE> can also be invoked as <CODE>B.transpose(A)</CODE>.
49 The latter may be more efficient, since Matrix may be able to re-use the storage of
50 A (if it has approximatly the right size and isn't currently shared with another matrix).
51
52 @sa G3D::Matrix3, G3D::Matrix4, G3D::Vector2, G3D::Vector3, G3D::Vector4, G3D::CoordinateFrame
53
54 @beta
55 */
